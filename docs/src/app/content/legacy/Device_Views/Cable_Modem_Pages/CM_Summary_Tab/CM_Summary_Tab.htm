CM Summary Tab

The Cable Modem Summary tab displays information about the cable modem including the MAC Address, IP Address, Ping Status, Vendor Information and RF Score. This page also contains an Action bar for performing diagnostic or troubleshooting actions on the modem.

For additional information on page elements, see Navigating CPE Device Pages.

For information on available actions, see CPE Device Actions.

The Cable Modem Summary displays the following data.

Panel

Displays . . .

Modem State

Overall status of the modem including IP Address, FQDN of the CMTS in the Family Tree of the device, Init state, Ping State, uptime, current lease end information, and whether the modem is in quarantine.

For additional information on Ping State, see below.

Signals

Upstream and Downstream Transmit, Receive, and SNR power information.  Items in this panel auto-refresh every 5 seconds. Click the Pop-out Window icon () to view data in a separate window.

Health

Health status bar for the modem. Select the number of days/weeks you want to view.

Provisioning State

Provisioning status for features. This panel refreshes automatically when the status changes from “offline” to “alive” mode.

Component Status

Lists any child devices and wireless gateways and the Ping status.

Click the wrench icon () to set up bridge mode and manage advanced wireless gateway.

Note: You need special user permissions to perform this function in RIO.

Bandwidth

Up and down Bandwidth, Burst and Priority information on the cable modem. This panel refreshes automatically when the status changes from “offline” to “alive” mode.

Spectrum Analyzer

Link to view Spectrum Analyzer data for the modem.

Note: This panel appears only if the device supports Spectrum Analyzer.

DOCSIS

DOCSIS Version, DOCSIS Capability, QOS Version, TFTP Server, and RIP Network

Hardware

Make, Model, Hardware Version, Software Version, Boot Version, and Serial Number

Firmware Status

Firmware file, version, and status information. This panel auto-refreshes every 5 seconds.

Flaps

Information about Flaps (e.g. when the modem goes offline and then comes back online ); including the number of Flap Hits (you can reset this to zero; see Clear Flaps), the most recent flap, and the CMTS interface.

Hover over each metric to view a pop-up definition of the metric.

Note: If the modem does not show Flap data, the modem's Object Identifiers (OIDs) are not working in the device.

Bridged Devices

Devices connected to the modem, if known.

 – If the MAC Address appears in blue, it is hyperlinked to another view.
  Click the link to go to the DHCP Summary page.

– In some cases the MAC Address is not hyperlinked and appears in black.

– If no bridged devices found, RIO displays a straight line.

DSG State

ADSG tunnel status and data relating to Timers, Timeouts, MAC Addresses, IP addresses, etc.

 

Modem Ping State

The Ping State is the ping response on the device. If returned, this shows the ping response in milliseconds. Other states are Unreachable or Unused.

In cases where a modem has two leases on the same CMTS, and only one is active, RIO lists other lease as Unused rather than showing it as a bad lease (a.k.a. Unreachable).  An example of this is when a modem has an IPv4 lease, then switches to an IPv6 lease.  The modem is only reachable on the IPv6 address, but that doesn’t mean the IPv4 lease is bad, it’s just not in use.  In this case the IPv4 lease is listed as Unused instead of Unreachable.  An example is shown below.
