Map Tab

In the Network Diagnostics module, RIO provides an interactive Map tab that displays status information for the entity level (Site down to Node) you are viewing. Depending on the level you are viewing, RIO provides different Map details.

The Map allows you to see areas experiencing difficulties, so you can use them as a troubleshooting aid as well as to identify issues before a customer calls.

To view a Map, select the Map tab in an entity View or select Map from the Service Path drop-down menu. A sample Hub map is shown below.

Each Map displays the children related to the selected entity.  For example, a Hub map (shown here) displays all Nodes within the Hub; and a Node map displays Accounts within the Node, etc. These entities are color coded depending on the selected Metric.

Click an entity to view details about the selected Metric in a pop-up scorecard.

Click the entity hyper-link to jump to that entity's map.

On a Node map, clicking the Account hyper-link takes you to the Account Summary page.

Use the standard map controls to zoom in or out on the map and to pan vertically or horizontally across the map. You can also toggle between Satellite and standard Map view.

Use the Filter field to filter data associated with the map. For example, you can filter on a street name, or a status color for a selected metric (see Filters).

The Map provides an option to view entities and Options in table format, click Show Table. A sample is shown below.

The table includes a column for the Metric selected and also color codes and filters items depending on selected Heatmaps and Filters.

Using Map Options

The Map provides options for viewing Metrics, GIS Data, Heatmaps, and other Filters; however, available options vary depending on the entity level you are viewing.

The following table defines available options.

Option

Description

Metrics

Metrics include Sign On Status, Signal Flux, Transmit and Receive Levels, and Signal to Noise Ratio. You can select only one metric at a time. The marker is color coded based on the selected metric.

Note: All maps include Sign-on Status. Additional metrics are available only on the Node and Account level maps.

Layers

The HFC Plant overlay displays “line art” representing TWC’s physical plant (fiber and coax cable). This information is populated from the GIS system.

Heatmaps

Heat Maps include Signal Flux, Receive Levels, Sign On Status, Signal to Noise Ratio, and Transmit Levels. You may select more than one Heat Map at the same time. The Heat Map color codes the area around the marker to show the relative health based on the selected layer(s).

Note: Heatmaps are available only on the Node and Account level maps.

Filters

The map contains a free-form Filter (similar to a "search" tool), and Device Filters (available at the Node level) that allow you to view only accounts that have selected devices.

 

Metrics

The entities shown on the Map (e.g. Hubs, Nodes, etc.) are color coded by the selected Metric to show the overall health of the entity for that Metric. Each device type is rated separately, with RIO displaying the color for the worst-case scenario. RIO displays the same threshold levels as in Unified or NYROC.

To view more detailed information, click the entity icon on the map. RIO displays a pop-up scorecard for the selected metric. An example for Account Sign-on Status is shown below.

The colors depict the following general status.

Color

Threshold

Green

Levels are "good."

Yellow

Levels are at a "warning" state.

Red

Levels are at a "critical" state.

By default, maps display Sign-on status when you first access them. Selecting a different metric refreshes the map and changes the color coding to represent the entity's health for that metric. The available Metrics for Node and Account are listed in the following table.

Metric

Shows . . .

Sign-on

Sign-on percentage for CPEs. This is the default metric.

Flux

Devices experiencing Transmit or Receive signal level fluctuation.

Transmit

Transmit power levels (TX) for the device. These must be within a certain range.

Receive

Receive power levels (RX) for the device. These must be within a certain range.

SNR

Signal to Noise Ratio (SNR) levels for the modem. These must be within a certain range.

Trouble Calls

Shows Accounts with active trouble calls and the corresponding Work Order number.

You can select only one Metric at a time. The marker is color coded based on the selected Metric. You may, however, select a Metric plus Heatmaps, and the GIS Overlay at the same time.

Layers

The Map provides options to view physical plant information and weather data. In many areas, at the Node and Account levels, map “line art” is available representing TWC’s physical plant (fiber and coax cable). This information is populated from the GIS system.

To view the GIS line art, select HFC Plant in the Layers list. A sample Node with line art turned on is shown below.

GIS information is not available in all areas.

Red represents coax and blue represents the fiber into the node.

To turn off line art, deselect the HFC Plant option in the Layers list.

The Layers option also includes weather radar data. To view, select NEXRAD.

 

Heatmaps

The heatmap color codes the area around the entity marker to show the relative health of the entity based on the selected map. The farther the levels are out of range or above/below threshold levels, the deeper the color. If you select more than one Heatmap, the color coding is cumulative. Heatmaps for Sign On and IVR Calls are available on all Maps; however, the Node map contains additional Heatmap options.  

Select the option(s) you want to view from the Heatmap list. A sample Hub heatmap for IVR Calls is shown below.

Heatmap options are defined in the following table.

Heatmap Option

Shows . . .

Sign-on

Accounts with some (yellow) or all (red) devices offline within the past few minutes.

Flux

Fluctuation in signal health over a given day. Deeper colors reflect more fluctuation.

Transmit

Transmit power levels (TX) for the device. These must be within a certain range.

Receive

Receive power levels (RX) for the device. These must be within a certain range.

SNR

Signal to Noise Ratio (SNR) that should be above a threshold for DOCSIS devices.

IVR Calls

Areas with calls to the Interactive Voice Response (IVR) system.

You may select more than one Heatmap at the same time, plus you may also select a Metric and GIS Overlay at the same time.

Filters

The Map contains two types of filters:

The Filter field is a text field that allows you to filter data associated with the map. For example, you can filter on a street name, or a status color for a selected metric.

Device filters are checkbox selections that allow you to view only accounts with associated devices.

Filter Field

To use the free text Filter field, enter your criteria and press [Enter].

A sample Node Map filtering on the color “red” is shown below. In this example, all accounts with a red Sign On status are shown on the map and listed in the table.

If you select a different metric from the Options list, the map displays results for that metric.

Additional filters may include street name, account number, etc.

Device Filters

The Device Filter (available at the Node level) allows you to view only accounts with the selected associated devices. Options include WiFi, Set-top, Cable Modem, and MTA. You can also select options to see which accounts have services offline, including All Service Outs, or by device type.

WiFi is not available in all areas.

To view a list of Account details, click Show Table.
