Using the Service Path Sidebar

The Service Path shows entity relationships. RIO displays information in a hierarchical path of the critical elements necessary for a service. In the Network Diagnostics module, the Service Path displays service-related information from the market level down to the subscriber account and device.

For example, if you search for a device (which is the lowest level), RIO displays the path from the device to the CMTS (or QPSK Demodulator, as appropriate) up to the market and enterprise levels.

From the Service Path, you can select to view other elements, which may assist in troubleshooting issues. For example, if viewing an Account you can easily view a summary of the Node by clicking the Node in the Service Path.

The Service Path appears as a sidebar on the left when viewing entities.

 

You can toggle the panel to hide or view it.

Hover over the Pin icon to view Service Path details. When you move the mouse, the Service Path closes. To keep the Service Path in view, click the Pin icon. To hide the Service Path, click the Pin again.

Service Path Entity Drop-down Menus

Each entity in the Service Path contains a drop-down menu. To view this sub-menu, click the arrow to the right of the entity. A sample is shown below.

Options listed in each menu vary by entity type.

The Service Path drop-down menu items display icons to indicate tabs currently in view.

The tag icon ()indicates the current default landing page.

The eye () indicates which tabs are currently in your view.

The grayed-out eye () indicates the tabs that are not currently shown in your view of the entity; however, you can still select them from the list.

You can modify your tab viewing preferences by entity type (see Changing Page Viewing Preferences).

Orphaned Devices

If RIO cannot determine the market in which a device belongs, RIO displays a message in the Service Path, to indicate the device is Orphaned.
